Side panel · live trip timeline · straightened

A persistent left input column collects route → flight + date → pax. The right canvas tells the trip story — empty at first, then a live timeline once a flight is locked in. The original six-step disclosure has been folded into four linear screens, with the rarer branches (no flight #, no match) shown as inline links rather than separate paths.

v2 · simpler
Visibility of system status Recognition over recall Minimise steps The flow is now Landing → Flight details → Confirm → Cars. Date and flight number share a single screen — a confident user fills both at once; an unsure user still gets one progressive cue (matches only appear once date is set). The right-canvas timeline persists from confirmation through to results so timing stays the centrepiece, never a payoff at the end. URLs change on real step changes; flight selection is in-page state inside /rides.

No flight # · branch

Inline fallback, never a screen of its own

  • Quiet link in screen 2 left panel: "I don't have my flight details yet".
  • Replaces the date + flight rows with a terminal + pickup time picker, in place.
  • Right canvas shrinks to a 2-row timeline (driver → destination) with a soft hint.
  • "Add flight later" promise stays: opening the booking after boarding-pass switches delay tracking on.
Banner: "You can add the flight later — we'll start tracking delays as soon as you do."

N → A drop-off · diff

Same shell. Departure flight. Backwards math.

  • Pickup = home/hotel; drop-off = airport (butter chip).
  • Screen 2 asks for the departure flight, same combined date + flight # layout.
  • Pickup time computed backwards: departure − (check-in + drive + buffer). Always show the math.
  • Timeline reverses: home → driver → drive → curb-side T5 → check-in cutoff.
Banner: "Departure 14:30 — we'll pick you up at 10:55 so you reach T5 by 11:35 (2h 55m before)."

N → N normal ride · diff

Skip screen 2 entirely.

  • Same URL /rides. Flight + date fields never appear.
  • Landing's left panel adds a quiet "Ride now / Schedule" toggle below pickup & drop-off.
  • Goes from screen 1 straight to screen 4 (cars). Right canvas is a 2-row timeline only.
  • Type an airport later and we slide screen 2 back in — same page.
Path summary: trips-card flight → straight to cars · trips-card hotel → one field prefilled · airport typed later → screen 2 grows in.

What's still open

  • Default sort on the car list — recommended vs price?
  • How loudly do we surface delay-tracking promise on landing?
  • Should "Hourly" live as a tile here or as a sibling product page?
  • Tablet — does the two-column shell collapse to a single column or keep both?

Why the simpler flow

  • Date and flight # in one screen: fewer screens to traverse for a confident user.
  • The skip / no-flight branch is real but rare — it lives as an always-visible link, not a screen of its own.
  • The right-canvas timeline stays the centrepiece across every step.
